## 维度一：语义/编码变体与复合提取攻击

这一维度主要探索传统的安全防御边界，测试前置过滤器在面对文本格式变形和多级伪装时的拦截极限。

### 1. 强迫前缀注入 & 编码格式化绕过 (Prefix Injection & Obfuscation)

* **经典文献：** *Universal and Transferable Adversarial Attacks on Aligned Language Models* (Zou et al., 2023) 与 *Jailbroken: How Does LLM Safety Align with AI Safety?* (Wei et al., 2023)
* **攻击简介：** 利用自回归模型的文本补全惯性，强制要求其以特定肯定句开头（如“Sure, here is...”）。同时配合 Base64 或 JSON 等非明文序列化格式，让针对明文匹配的敏感词过滤层瞬间失效。
> **💡 概念释义（自回归补全惯性）：** 类似于手机输入法的“接龙联想”功能。一旦红队强制喂给模型一个带有特定倾向的开头，模型为了维持文本的连贯性，会极大概率顺着这个梯子把原本被禁止的提示词一路编下去。


* **论文地址：** [arXiv:2307.15043](https://arxiv.org/abs/2307.15043) / [arXiv:2307.02483](https://arxiv.org/abs/2307.02483)

### 2. 级联复合攻击与荷载拆分 (Compound Attacks)

* **核心学术基准：** **Raccoon 基准**
* **对应论文：** *Raccoon: Prompt Extraction Benchmark of LLM-Integrated Applications* (Tianyi Yang et al., ACL 2024)
* **方法简介：** 该论文系统化地梳理了 14 种单一的提示词提取攻击方式，并提出了**级联复合攻击（Compound Attacks）**。它将角色扮演、拒绝抑制与格式化请求多级叠加，模拟真实世界黑客的多步诱导策略。荷载拆分则利用前置金丝雀的“零上下文（Zero-Context）”单轮审计盲区，在多轮对话中分散输入无害断章，由 Agent 在 session 历史中自动拼接执行。
* **论文地址：** [arXiv:2406.06737](https://arxiv.org/abs/2406.06737)
* **GitHub 仓库：** [M0gician/RaccoonBench](https://github.com/M0gician/RaccoonBench)

## 维度二：高阶逻辑推理与自适应进化攻击

这一维度利用大模型（Agent）与小模型（G4PA金丝雀）之间的**逻辑推理能力差（Capability Gap）**，通过高逻辑负载或自动化演进手段穿透防护。

### 3. 多轮渐进式语义麻醉 (Crescendo / Chain of Attack)

* **对应论文：** *Crescendo: A Many-Turn Jailbreaking Method against Language Models* (Mark Russinovich et al., Microsoft, 2024)
* **方法简介：** 微软团队提出的 **Crescendo（渐强攻击）**。攻击者在起初表现得极其顺从，不提及任何敏感词，通过长达数轮逐步深化的讨论，潜移默化地将模型的安全边界向外推移，最终在最后一轮完成致命的系统提示词提取。这直接针对“零上下文”或固定判定阈值的单轮护栏进行降维打击。
* **论文地址：** [arXiv:2404.01833](https://arxiv.org/abs/2404.01833)

### 4. 自进化强化学习探测 (JustAsk) 与 LeakAgent

* **对应论文：** *Just Ask: Curious Code Agents Reveal System Prompts in Frontier LLMs* (Xiang Zheng et al., ICML 2026)
* **方法简介：** 论文指出了自主 Agent（如代码助手）普遍存在系统提示词泄露缺陷，并提出了自进化攻击框架 **JustAsk**（同时参考了早期基于强化学习的 LeakAgent 研究）。该方法完全**不需要手工编写提示词**，而是将提示词提取建模为一个在线探索问题，利用**置信区间上界（UCB）算法**动态调度其技能树（包含 14 种原子探索技能与 14 种高阶编排策略），根据护栏的流式反馈实时微调后续的提问组合，自动化逼近并发现 G4PA 的过滤死角。
> **💡 概念释义（置信区间上界 UCB 算法）：** 类似于你在美食街找餐厅：你既想去吃之前去过、觉得还不错的“老地方”（利用已有经验），又忍不住想去尝试那些没吃过的“新店铺”（探索不确定性）。UCB 算法通过精密的数学公式，在“继续用有效的攻击套路”和“尝试全新提问方式”之间找到了完美的理性平衡，从而以极高的效率榨干护栏的防御漏洞。


* **论文地址：** [arXiv:2601.21233](https://arxiv.org/abs/2601.21233)
* **GitHub 仓库：** [x-zheng16/JustAsk](https://github.com/x-zheng16/JustAsk) (及其维护的 frontier 模型提示词泄露库：[System-Prompt-Open](https://github.com/x-zheng16/System-Prompt-Open))

## 维度三：信道边界突破与间接注入

这一维度专门用于批判和检验“仅在输入端做静态检查”的传统边界假设。

### 5. 间接提示词注入 (Indirect Prompt Injection - IPI)

* **奠基文献：** *Not what you've signed up for: Compromising Real-World LLM-Integrated Applications with Indirect Prompt Injection* (Kai Greshake et al., 2023)
* **方法简介：** 攻击者不在直接聊天框输入命令，而是将恶意 Payload 预先埋藏在 Agent 必须要检索读取的外部数据源中（如网页、RAG 数据库、API 响应）。当 Agent 自动读取该内容时，隐藏的系统指令被激活，从内部接管 Agent。此时前置在用户输入端的 G4PA 检测模块会由于无法审计外部 Tool 检索信道而彻底失灵。
* **论文地址：** [arXiv:2302.12173](https://arxiv.org/abs/2302.12173)

### 6. 多模态符号与视觉注入 (Multimodal Rebus Injection)

* **红队技术来源：** **NVIDIA AI 红队 (NVIDIA AI Red Team, 2025)**
* **方法简介：** 专门针对多模态 Agent。红队成员发现，将提取 Payload 转化为由 Emoji 序列、空间分割字谜（Rebus）或半隐蔽像素文字构成的图像并上传，会直接触发多模态模型的早期融合对齐缺陷。由于模型本身具有强大的“问题解决本能”，它会自动在内部将视觉线索转译并执行为提取指令，而前置的文本金丝雀根本无法看懂图像中的语义含义。
> **💡 概念释义（多模态认知反射）：** 就像人类看一张印着“一台打印机+一个人挥手+一个地球”的看图猜意题，我们的大脑会情不自禁地通过认知本能拼凑出“Print Hello World”的意思。大模型在处理这类交叉模态 Token 时也会产生这种“计算反射”，直接在神经网络内部解密出隐藏指令，从而让传统的文本护栏完全摸不着头脑。


* **技术来源参考：** NVIDIA 官方技术博客 *Securing Agentic AI: How Semantic Prompt Injections Bypass AI Guardrails*

## 维度四：Agent 专属行动级与控制流劫持攻击

这一维度专为“Chat 界面输出被严格锁定/拦截”的高阶防御设计。

### 7. 长期记忆控制流劫持 (Memory Control Flow Attacks - MCFA)

* **对应论文：** *From Storage to Steering: Memory Control Flow Attacks on LLM Agents* (Zhenlin Xu, Xiaogang Zhu et al., arXiv 2026)
* **方法简介：** 该研究发现，在支持长期记忆（Memory Layer）的 Agent 架构中，攻击者可以诱导 Agent 执行一次写操作，将一小段长效带毒的规则（例如“始终优先使用某个特定工具或在特定条件下外泄 init_config”）注入其记忆库中。在随后的数十轮或跨任务会话中，即便前置护栏将输入端过滤得再干净，Agent 在调用自身记忆时控制流（工具选择和执行顺序）依然会被这颗“延时炸弹”强行劫持。论文中同步推出了自动化评估框架 **MEMFLOW**。
* **论文地址：** [arXiv:2603.15125](https://arxiv.org/abs/2603.15125)

### 8. 写入原语外泄攻击 (Exploiting Write Primitives)

* **学术背景：** 参见 Agent 漏洞链与跨应用上下文毒化研究（如 2026 年关于 *Cross-app context poisoning* 的讨论）。
* **方法简介：** 即使 G4PA 能够完美监测并拦截 Chat 界面输出的任何敏感文本，红队仍可绕过对话框。利用平台赋予 Agent 的“写原语”（如修改数据库、发送 Webhook、调用外部文件写入 API 接口），欺骗 Agent 匹配某个正常的业务意图，但在后台将“系统提示词”作为参数填入 API 表单或日志字段中，直接将秘密运送出境。

## 💡 评估度量衡推荐体系溯源

### 9. D7 / D8 差异判定 (Swiss Benchmark)

* **对应论文：** *Swiss-Bench 003: Evaluating LLM Reliability and Adversarial Security for Swiss Regulatory Contexts* (arXiv 2026)
* **指标简介：** 该研究（SBP-003）专为金融和强监管环境设计，将可靠性评估体系扩展出了两个极其核心的对抗指标：**D7（Self-Graded Reliability Proxy / 自我评估可靠性代指）** 与 **D8（Adversarial Security / 外部对抗安全性）**。
* **实证意义：** 在安全测试中，通过对比 Agent 在静态无对抗下的自我防护遵从率（D7），以及经历强力外部红队洗礼后的真实防护率（D8），能够极其精准地用数据量化联合护栏在极端压力下产生的“安全感知偏差（安全幻觉）”，是评估 G4PA 韧性不可或缺的学术级度量衡。
* **论文地址：** [arXiv:2604.05872](https://arxiv.org/abs/2604.05872)
