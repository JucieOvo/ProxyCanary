# Guard4PromptAttack 后续投稿策略

## 论文当前定位

- **类型**：新范式提案 / 可行性验证（非成熟系统论文）
- **优势**：概念创意（攻击识别 vs 攻击有效性验证）、自我批判诚实、叙事清晰
- **劣势**：实验规模小（54 攻击、单模型族）、核心假设未验证（行为传递）、基线偏弱
- **综合估分**：约 2.5-3.0 / 5.0

## 投稿策略总纲

**核心原则：不伪装成顶会系统论文，以「提出一个好问题 + 给出初步诚实答案」为卖点。**

三条路径，按推荐度降序：

---

## 路径 A（推荐）：TMLR 开放评审

**TMLR（Transactions on Machine Learning Research）**

- **Rolling submission**：无 deadline 压力，随时可投
- **开放评审（Open Review）**：审稿意见公开，不会因单审稿人偏好被无理由拒稿
- **允许 revision 循环**：被要求修改后可以补实验重新提交，不被直接 kill
- **重视诚实**：社区对「承认局限并诚实讨论」的论文友好
- **接收范围广**：从理论到应用、从小规模到大规模均可
- **认可度**：在 ML/NLP 社区逐渐被视为 ICML/NeurIPS 的轻量替代

**如果被要求修改**：可在 revision 期间补 Qwen 0.6B ↔ 9B 交叉实验（行为传递的初步证据），通过率大幅提升。

---

## 路径 B：ACL/EMNLP Findings

**ACL 2027 或 EMNLP 2027 Findings**

- Findings 接受「有趣的初步探索」，对实验规模容忍度高于 main
- 自我批判姿态在 Findings 审稿中可能是加分项——审稿人不会要求你补一个你坦诚没做的实验
- 煤矿金丝雀的 intuition 是可传播的，容易被记住
- 页数限制（通常 4-6 页正文）与本文当前体量匹配

**建议提交前准备**：
1. 压缩到 Findings page limit（约 6 页正文 + 参考文献）
2. 进一步突出 concept 部分，精简实验细节
3. 考虑加一小节「如果行为传递假设为真/为假的两种情况分析」——比补实验更便宜且展示思维深度

**同时投稿 Workshop**（被拒后的保险）：
- **TrustNLP @ NAACL/ACL**：LLM 安全与可信赖 workshop
- **Safe Generative AI @ NeurIPS**：安全 + 生成式 AI
- Workshop 接受率通常 30-50%，远高于 main

---

## 路径 C：安全类期刊/会议

**Computers & Security (COSE)**
- CCF B 类 / SCI 二区（Elsevier）
- 青睐具体的安全防御系统和架构设计
- 对实验规模要求低于顶会，更重工程实用性

**IEEE Access**
- SCI 三区（巨型 OA 期刊）
- 审稿极快（4-6 周一审），对资历无限制
- 适合急需发表的情况

**SecureComm / NSS**（CCF C 类会议）
- 偏向系统安全，与架构级阻断的定位契合
- 可投 Short Paper（4-5 页）

---

## 不推荐的投稿目标

| 刊物 | 原因 |
|------|------|
| ACL/NeurIPS/ICML main | 实验规模完全不够，desk reject 概率高 |
| IEEE S&P / USENIX Security main | 需要完整威胁模型形式化和安全性证明，本文不具备 |
| ICLR | 偏重方法论创新，本文方法较简单 |
| Nature Machine Intelligence | 需要里程碑式贡献，远未达到 |

---

## 可做的「低成本增强」（不需重做大规模实验）

按投入产出比排序：

1. **Qwen 0.6B ↔ 9B 交叉实验**（1-2 天）
   - 0.6B 作金丝雀，9B 作受保护模型，量测 TPR/TNR
   - 9B 作金丝雀，0.6B 作受保护模型，量测 TPR/TNR
   - 直接回应「行为传递假设」，是投入产出比最高的实验

2. **「理论分析：行为传递的条件」小节**（1 天）
   - 不补实验，但给出形式化分析
   - 在什么条件下金丝雀行为能预测受保护模型的行为
   - 引用对抗可迁移性文献（Papernot 2016, Tramèr 2017）的理论框架
   - 展示有能力框架化这个问题，即使暂缺实验

3. **开源代码仓库**（半天）
   - README + 复现指南
   - arXiv 预印本 + GitHub 链接

```
当前 (2026.06)
    │
    ├─→ 1 周：补 Qwen 交叉实验（可选但强烈建议）
    │
    ├─→ 2026.07：投稿 TMLR（rolling，随时可投）
    │   或 → 等待 EMNLP 2027 Findings deadline
    │
    ├─→ 审稿期间：如被要求补实验，则补充跨模型数据
    │
    └─→ 同步：arXiv 预印本（投稿同时或稍早）
```

---

## 最终建议

**优先 TMLR**：rolling submission 无 deadline 压力，开放评审降低随机性，revision 期间可补实验。

**如果急需发表**：IEEE Access（4-6 周见刊）或 COSE。

**如果追求学术声望**：先补 Qwen 交叉实验，再投 ACL 2027 Findings。
