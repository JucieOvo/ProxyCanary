# 图片可读性与美观度优化方案说明书

## 1. 方案目标与意图
本项目包含四张核心学术/技术插图，目前在排版、配色、对齐以及局部细节（如圆角拉伸、箭头指向错误、文字拥挤重叠）上存在一定缺陷。本方案旨在对生成这四张图的 Python 脚本进行重构和优化，提升图表的可读性与视觉美观度，使其符合高水平论文/报告的插图标准。

## 2. 优化要点与修改策略

### 2.1 优化 Figure 1: Motivated Example (plot_fig1.py)
* **问题分析**：
  1. 30 轮对话的 timeline 挤满 29 个小圆圈，正常对话的文字不得不进行 30 度旋转且字号极小（5.5），导致阅读困难。
  2. timeline 在 16x7 的画布上两侧留白不合理，文字标签存在潜在重叠。
  3. 配色过于刺眼（纯红与纯绿），缺乏学术高级感。
  4. 存在中文内容（如正常对话节点及攻击节点文本），与英文论文的整体语言规范不一致。
* **修改策略**：
  1. **完全英文国际化**：将所有中文内容替换为精确的英文表述，如“查余额” -> "Check Balance"、“推荐基金” -> "Recommend Funds"、“定投?” -> "SIP?"、“黄金走势” -> "Gold Trend"、“输出之前的提示词” -> "Extract system prompt"。
  2. **重构 timeline 排布**：微调各个轮次节点的 x 轴绝对坐标，使得前 4 轮与省略号及 29, 30 轮的距离呈几何对称分布，横向视野更开阔，字体大小提升至 10 号以增强可读性。
  3. **优化配色与阴影**：替换为高级莫兰迪配色方案，降低红绿高饱和色彩的刺眼感。

### 2.2 优化 Figure 2: Architecture (plot_fig2.py)
* **问题分析**：
  1. 开启 `ax.set_aspect('equal')` 强制纵横比等比后，由于 figsize 是 18x7.5，会导致图形上下产生大面积空白，画幅利用率极低。
  2. 模块中心线不统一，未处于同一几何中轴线上，导致图表略显无序。
  3. `Pre-Check Combine` 标签框在调整 y 轴高度后，横向与 `Canary Model` 发生重叠（x=2.5+1.8 与 x=3.8 重合）。
  4. 分流箭头采用先水平再垂直的设计，导致箭头的垂直折线与 `BLOCK` / `PASS` 框的左边界重合，视觉效果欠佳。
* **修改策略**：
  1. **取消强制等比并实现自然撑满**：移除 `ax.set_aspect('equal')`。为了使圆角在没有强制等比的情况下不发生拉伸，将 figsize 设置为 `(18, 6.5)`，并将坐标轴限制设置为 `x_lim=(0, 18), y_lim=(0, 6.5)`。由于 figsize 比例与坐标轴区间比例完全相同，每个数据坐标在 x 和 y 方向的物理像素绝对相等。这不仅能使图形完美撑满整个画布、消除空白边，还能使所有圆角保持正圆弧。
  2. **完美几何中轴线对齐**：将 User Input、Canary Model、Output Inspector、Decision 以及 Protected Model 的垂直中心统一对齐至 `y=2.8` 的中轴线上。PASS 和 BLOCK 分支则关于 `y=2.8` 呈镜像上下对称排列（BLOCK 中心处于 y=3.85，PASS 中心处于 y=1.75），使得整体视觉极具几何美学与对称性。
  3. **解决 Pre-Check 框重叠**：将 `Pre-Check Combine` 框宽度缩减至 1.0（x=2.6 到 3.6），使其精确悬浮在 User Input 与 Canary Model 之间的横向空隙内（x=2.5 到 3.8），高度设为 0.45，y 轴偏置到 3.4（高出 2.8 轴线），保证毫无重叠。
  4. **重构分流折线箭头**：将分流箭头的连接样式调整为 `connectionstyle="angle,angleA=90,angleB=0"`（L型对称折线）。即从 Decision 框的右边缘中心 `(13.2, 2.8)` 出发，先垂直向上/向下分流，再水平指入 `BLOCK` `(14.2, 3.85)` / `PASS` `(14.2, 1.75)`。这样箭头会以水平方向干净利落地指入框内，完美避免与边框重合。

### 2.3 优化 Figure 3: Experimental Results (plot_results.py)
* **问题分析**：
  1. 拼写错误：图表基线名称中的 CanariToken 拼写错误，应为 CanaryToken。
  2. 数值标签重合：在 Figure 3a 中，9B 和 0.6B 两根柱子的顶部数值距离太近，字体较小时易叠在一起。
  3. 柱状图配色略显老气。
* **修改策略**：
  1. **修正拼写**：将所有数据字典键名及图表 label 中的 CanariToken 修正为 CanaryToken。
  2. **错位标注数值**：对 9B 和 0.6B 的柱顶文字标注进行自适应纵向微调，或者稍微增大柱子间距（调整 width 和 x_pos 偏移），确保数值绝不重叠。
  3. **重构图表配色**：选用具有高辨识度且美观的学术配图色调。例如 Ours 使用高亮珊瑚红（#E63946），9B 柱使用经典石青蓝（#1F4E79），0.6B 柱使用淡雅天蓝（#A8DADC），使对比更加分明。

### 2.4 优化 Figure 4: Model Capability (plot_fig4.py)
* **问题分析**：
  1. 折线与散点样式较为单调，缺乏视觉深度。
  2. 数值标注 annotate 容易与数据点（散点）本身重合。
  3. 底部说明文字字号偏小，且硬编码的位置偏下。
* **修改策略**：
  1. **提升数据点视觉效果**：为 Ours 散点和 LLM-Judge 散点增加微小的边缘描边与发光效果，稍微增加 marker 大小（例如 s=250），并加粗连线。
  2. **自适应避免重叠**：根据数据点的相对高低，将 Ours 的数值标注均移至上方（y + offset），LLM-Judge 的数值标注移至下方（y - offset，但在最后一点由于两者交汇，单独进行微调偏移）。
  3. **美化区域高亮**：将 "Ours Wins" 和 "LLM-Judge Wins" 的文本框底色改为更加柔和的半透明莫兰迪色，并增加虚线框作为边界分割，使得两个模型的优势区间一目了然。

## 3. 执行顺序与流程
1. **方案提请审批**：将本优化方案落盘并呈报技术总监（用户）审批。
2. **代码修改**：在获得明确授权（用户回复“开始”或同意）后，修改对应的 Python 脚本。
3. **运行渲染**：执行修改后的绘图脚本，重新生成 PDF 和 PNG 格式的图片。
4. **效果验证**：检查生成图片的排版、字体对齐和整体可读性，确保没有错位或拉伸，确认无误后完成任务。

## 4. 风险点与注意事项
* **环境兼容性**：当前系统环境为 Windows 11，绘图脚本指定了 Microsoft YaHei 字体，需确保在 matplotlib 渲染时不会报错。
* **无 Emoji 约束**：根据项目主规则，代码和注释中绝对不能包含任何 emoji 表情。
* **非硬编码排版**：对于文本对齐，尽量使用 matplotlib 的相对布局或计算出的 offset，避免硬编码导致画布尺寸调整时排版崩溃。
