# Guard4PromptAttack 研究路线图

## 第一篇：金丝雀试毒的可行性验证（当前论文）

**核心问题**：攻击能不能得逞？

**方法**：独立金丝雀模型 + 双路径标记词 + 流式子串匹配

**贡献**：
- 区分「分析输入话术」vs「观察模型行为」两个检测对象
- 金丝雀试毒机制——将标记词嵌入金丝雀提示词，受保护模型零修改
- Raccoon 基准上的初步但诚实的实验证据
- 坦率承认行为传递假设未验证，定位为可行性验证

## 第二篇：话术特征与攻击识别率的因果关系（另一个项目）

**核心问题**：攻击为什么被理解为攻击？什么情况下模型将套取话术理解为攻击意图，什么时候会忽略攻击去执行无关内容？

**方向**：
- 从语言学/语义学角度分析套取攻击话术的形式特征
- 话术措辞（直接 vs 间接、命令式 vs 建议式、角色扮演 vs 直白请求等）与攻击识别率之间的规律
- 哪类措辞容易触发模型的安全拒绝
- 哪类措辞容易让模型「跑偏」去执行无关内容
- 为金丝雀方法的漏检（约 24%）提供语言学解释

**与第一篇的关系**：补上第一篇中「层次一」（攻击识别）的理论基础——当前第一篇把层次一交给了已有方案，但「模型为什么把一段话术识别为攻击」本身是一个开放问题

## 第三篇：行为传递——金丝雀能预测受保护模型吗？

**核心问题**：金丝雀模型的行为反应在多大程度上能预测受保护模型的行为？

**方向**：
- 同一 Qwen 族内：0.6B / 1.5B / 7B / 9B 互相作为金丝雀和受保护模型，交叉测试
- 跨模型族：GPT-4o-mini / Claude / DeepSeek 作受保护模型，Qwen 作金丝雀
- 量测假阳性拦截率（金丝雀拦截但受保护模型不会被攻破）和假阴性放行率（金丝雀放行但受保护模型被攻破）
- 引用对抗样本可迁移性文献（Papernot 2016, Tramèr 2017）作为理论框架

**与第一篇的关系**：直接回答第一篇在 Section 5.3 中承认为未验证的核心前提

## 三篇论文的逻辑链

```
第一篇（当前）：立方法
  └─→ 金丝雀试毒机制可行，但有两个悬而未决的问题：

      ② 金丝雀的行为能代表受保护模型吗？

第三篇（行为传递）：验证边界 ②
  └─→ 跨模型可迁移性量化证据
      金丝雀→受保护模型的预测准确率
```

闭环：第一篇立方法 → 第二篇找根因 → 第三篇验证边界
